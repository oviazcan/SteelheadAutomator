# bulk-upload 1.3.2 perf + robustez Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) o superpowers:executing-plans para implementar tarea por tarea. Steps usan checkbox (`- [ ]`) para tracking.

**Goal:** Reducir el tiempo de bulk-upload para 100 PNs de ~3-4 min a ~1.5-2 min (y proyectado para 9k Schneider de ~5 hrs a ~2 hrs) eliminando llamadas espurias (strip1 unique-constraint, AddParams 500s espurios), paralelizando STEP 6b, y endureciendo el resume para que un atorón a media corrida no obligue a empezar de cero.

**Architecture:** Cambios contenidos a `remote/scripts/bulk-upload.js` (IIFE de un solo archivo) + `remote/config.json` (bump version). Sin schema GraphQL nuevo, sin hashes nuevos. Patrones existentes reusados: `runPool`, `withRetry`, `existingPnFullCache`, `findExistingQuote`, `setPanelPhase`/`setPanelProgress`.

**Tech Stack:** JS vanilla, MV3 Chrome extension main-world inyectado, Apollo persisted queries vía cookies.
