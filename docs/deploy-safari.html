0

Instalar Xcode

Una sola vez

Hoy la Mac solo tiene Command Line Tools; el converter necesita Xcode completo.

  1. App Store → instala Xcode (~7 GB).
  2. Apúntalo y acepta la licencia:
sudo xcode-select -s /Applications/Xcode.app/Contents/Developer
sudo xcodebuild -license accept
  1. Verifica que el converter exista (debe imprimir una ruta):
xcrun --find safari-web-extension-converter
1

Generar el proyecto Xcode

Con el converter, desde la carpeta ya armada

cd /Users/oviazcan/Projects/Ecoplating/SteelheadAutomator
xcrun safari-web-extension-converter safari/extension \
  --project-location safari/xcode \
  --app-name "Candado Surtido" \
  --bundle-identifier com.ecoplating.candadosurtido \
  --ios-only \
  --copy-resources \
  --no-open
  • --ios-only: solo target iPad/iPhone (no macOS). Si luego quieres probar en Mac, regeneras sin este flag.
  • --copy-resources: copia los .js/manifest dentro del proyecto (autocontenido). Ver Mantener al día por el tema de divergencia.
  • Al terminar imprime un resumen.

⚠️ El converter advierte world … not supported — es esperado: el validador del converter no reconoce la clave; Safari reciente sí la soporta en runtime y el proyecto se genera igual. No cambies nada todavía: termina el build (Pasos 2-3) y prueba en el iPad (Paso 5). Solo si NO intercepta (el _getState no se puebla) pasa al plan B. El warning por sí solo no prueba que falle.

2

Abrir y firmar

Con tu Apple ID gratis (sideload 7 días)

open "safari/xcode/Candado Surtido/Candado Surtido.xcodeproj"

En Xcode, panel izquierdo → selecciona el proyecto (ícono azul arriba). Verás 2 targets: Candado Surtido (la app) y Candado Surtido Extension (la extensión). Para CADA target:

  1. Pestaña Signing & Capabilities.
  2. Marca Automatically manage signing.
  3. Team → elige tu Apple ID (Personal Team). Si no aparece: Xcode → Settings → Accounts → “+” → Apple ID. No necesitas cuenta de pago para el POC.
  4. Bundle Identifier único. Si marca conflicto, cámbialo a algo tuyo, p. ej. com.TUNOMBRE.candadosurtido (la extensión queda …candadosurtido.Extension).

💡 Firma con Apple ID gratis vale 7 días: suficiente para el POC. Para producción se usa cuenta Apple Developer ($99) + TestFlight o MDM — ver el documento de decisión.

⚠️ Error común al hacer Run: “Embedded binary's bundle identifier is not prefixed with the parent app's bundle identifier.” El Bundle ID de la extensión debe empezar con el de la app + un sufijo. Fix: copia el del target de la app (p. ej. com.tu.candadosurtido) y pon el de la extensión = ese mismo + sufijo → com.tu.candadosurtido.Extension. Ambos targets con el mismo Team.

3

Instalar en el iPad

Por cable, desde Xcode

  1. Conecta el iPad por cable. En el iPad, toca Confiar (“Trust This Computer”).
  2. En Xcode, barra superior: destino = tu iPad (no un simulador), esquema = la app Candado Surtido.
  3. Product → Run (▶). Compila e instala la app contenedora en el iPad.
  4. Primera vez: en el iPad ve a Ajustes → General → VPN y gestión de dispositivos → [tu Apple ID] → Confiar.

4

Activar la extensión en Safari

En el iPad

  1. Ajustes → Apps → Safari → Extensiones (en iPadOS viejos: Ajustes → Safari → Extensiones) → activa Candado Surtido.
  2. Abre Safari → app.gosteelhead.com. Toca ᴀA / el ícono de extensiones en la barra → Gestionar extensiones → actívala y concede Permitir en este sitio (o Permitir siempre).
5

Probar el candado

El momento de la verdad

  1. Inicia sesión en Steelhead en Safari del iPad (valida que el login OAuth funciona con la extensión activa).
  2. Ve al board de surtido: /Domains/344/Workboards/6234, step “Preparando Surtido en Almacén”.
  3. Intenta mover una pieza NO programada → debe bloquearse (toast rojo).
  4. Una pieza programada → se mueve normal.

🔍 Diagnóstico (Web Inspector remoto)

Safari del iPad no tiene consola en el dispositivo. Para inspeccionar:

  1. En la Mac: Safari → Ajustes → Avanzado → “Mostrar funciones para desarrolladores web”.
  2. En el iPad: Ajustes → Safari → Avanzado → Inspector web → ON.
  3. iPad conectado por cable → en la Mac, Safari → menú Desarrollo → [tu iPad] → [pestaña de Steelhead].
  4. En la consola:
window.SurtidoGuard._getState()

Debe mostrar scheduled > 0 y surtido con el recipeNodeId. Si salen vacíos, el candado está en fail-safe (no bloquea) → revisa que los queries del board hayan cargado, o que world:"MAIN" esté interceptando (si no, plan B).

B

Plan B — si world:"MAIN" no funciona

Requiere iPadOS 17+; ya está listo en el repo

Si el _getState no se puebla o no bloquea, cambia al loader vía <script>-tag:

cd safari/extension
mv manifest.json manifest.world-main.json   # respalda variante A
mv manifest.fallback.json manifest.json      # activa el loader <script>-tag

Vuelve al Paso 1 (regenera con el converter) y al Paso 3. Para volver a la variante A, invierte el rename.

Mantener al día

Anti-divergencia

La fuente única del candado es remote/scripts/. Cuando cambie su lógica:

safari/sync-scripts.sh        # remote/scripts → safari/extension
# regenera el proyecto Xcode (Paso 1, con --force) y recompila (Paso 3)

tools/deploy.sh ya avisa si safari/extension/*.js quedó desincronizado de remote/scripts/. El proyecto generado safari/xcode/ es artefacto de build y no se versiona.
