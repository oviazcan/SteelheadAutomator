4. Estrategia de captura de mutations SIN afectar datos

① Captura pasiva cero riesgo · preferida

El scanner corre durante el uso normal. Cuando el operador ejecuta una mutation legítimamente (guardar un precio, crear una etiqueta), el scanner captura su hash de paso. No se afecta ningún dato extra — es una operación que igual iba a ocurrir. Cubre las mutations frecuentes solas, con el tiempo.

② Sentinelas (Fase C) reversibles

Para mutations raras y reversibles (Create, Update): un objeto de prueba dedicado marcado __SA_SENTINEL__. El ciclo: desarchivar → modificar/crear (dispara la mutation, captura el hash) → revertir/borrar. Salvaguardas: fail-closed si no es sentinela, blast-radius ≤ 1, journal idempotente para reparar si se interrumpe. Código ya listo (sentinels.mjs, mutation-runner.mjs), pendiente activar.

③ Destructivas nunca automático

Delete*, Archive*, Unset*: no se capturan con sentinelas automáticas. Solo captura pasiva (cuando el operador borra algo real) o manual supervisada. El autopilot las escala por correo si rotan, nunca las ejecuta.
