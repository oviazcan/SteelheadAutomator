Sembrado de centinelas — Fase C

Objetos canario que dejan al hash-autopilot regenerar el hash de una mutation rotada sin tocar datos reales.

⚠ POR QUÉ IMPORTA LA PRECISIÓN — El ciclo corre desatendido en producción. Si un centinela se confunde con un objeto real, el autopilot mutaría datos productivos. Las salvaguardas de código (isSentinel fail-closed, blast-radius, journal) son la última línea; el sembrado correcto es la primera.

Principios de seguridad — no negociables

  1. Nunca uses un objeto real. Crea uno nuevo dedicado, que no participe en ninguna operación.
  2. Marca inequívoca: el objeto lleva el literal __SA_SENTINEL__ en su nombre (o tags/customInput). Sin ella, el runner se rehúsa a mutar.
  3. Uno por tipo de entidad. El blast-radius es ≤1.
  4. En reposo = archivado. El ciclo lo desarchiva, muta y re-archiva; entre corridas siempre archivado.
  5. Sin dependencias. Nada ligado a datos reales (OV sin partes, PN sin inventario…).
  6. Solo mutations reversibles. Las destructivas (Delete/Remove) escalan — no se siembra centinela para ellas.

Procedimiento por entidad

  1. Identifica la entidad y una mutation reversible que la toque (ej. ReceivedOrder ← SaveReceivedOrderLinesAndItems).
  2. Crea el objeto canario en el ERP con la marca en el nombre: 
    ZZZ __SA_SENTINEL__ no-tocar (hash-autopilot)
    El prefijo ZZZ lo manda al fondo de los listados.
  3. Déjalo archivado (estado base).
  4. Regístralo en tools/hash-autopilot/sentinels-config.json: 
    {
  "entities": {
    "ReceivedOrder": {
      "id": "<idInDomain-o-nano-del-centinela>",
      "marker": "__SA_SENTINEL__",
      "baseState": "archived",
      "module": "ReceivedOrders"
    }
  }
}
  5. Verifica identidad: 
    node -e "import('./tools/hash-autopilot/sentinels.mjs').then(m => \
  console.log('isSentinel:', m.isSentinel({ name: 'ZZZ __SA_SENTINEL__ no-tocar' })))"
# → isSentinel: true

Mantenimiento & qué NO hacer

  • Tras cada corrida real, confirma que el centinela quedó archivado (la verificación post-ciclo alerta si no).
  • Journal con entrada dirty persistente → ciclo interrumpido; el siguiente run repara primero.
  • ❌ No marques un objeto real como centinela “temporalmente”.
  • ❌ No siembres centinela para Delete/Remove (escalan por diseño).
  • ❌ No dejes el centinela desarchivado ni con relaciones a datos reales.
  • ❌ No reutilices un centinela entre tipos de entidad distintos.
Recuerda: sembrar un centinela no dispara nada por sí mismo. El ciclo de una entidad solo corre si esa mutation rotó (lo marca el validator) — el centinela solo habilita la captura segura cuando haga falta.
